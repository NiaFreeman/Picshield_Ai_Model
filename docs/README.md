# Documentation Directory

This directory is for detailed project documentation.

## What to Put Here

- Architecture documentation
- API documentation
- User guides
- Development guides
- Research papers and references

## Example Structure

```
docs/
├── model_architecture.md    # Model design details
├── api_reference.md         # API documentation
├── user_guide.md           # How to use the system
├── development_guide.md    # For contributors
└── references.md           # Papers and citations
```

## Documentation Tips

- Use clear, simple language
- Include code examples
- Add diagrams where helpful (draw.io, mermaid)
- Keep documentation up to date with code
- Link to external resources

## Tools for Documentation

- **Markdown**: For simple docs (what we're using)
- **Sphinx**: For Python API docs
- **MkDocs**: For project documentation sites
- **Jupyter Book**: For interactive documentation
